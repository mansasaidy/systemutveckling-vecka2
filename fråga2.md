# Fråga 2

Ett Git-commit är som att spara en stegen av ditt projekt. Varje commit innehåller de ändringar du gjort och ett meddelande som förklarar vad du ändrade. Det är viktigt eftersom du alltid kan gå tillbaka till ett tidigare läge om något går fel. 

Om du jobbar på en hemsida och råkar göra en ändring som gör att sidan slutar fungera. Om du har committat tidigare kan du lätt backa tillbaka till den fungerande versionen istället för att försöka gissa dig fram till vad som blev fel. Git hjälper dig att inte förlora fungerande kod och gör det tryggare att experimentera.
